1. What are the different phases of React componentâ€™s lifecycle?

Answer: Three main phases :

  • 1)Mounting phase :

    • In this phase, constructor will work first and render , inside render there will be return and 'div', after that ComponentDidMount method will run, it will run only once.

  • 2)Updating Phase :

    • In this phase , methods like "shouldComponentUpdate", render(), "componentDidUpdate" are involved here. shouldComponentUpdate allows optimization by deciding whether a render should occur based on prop or state changes.

  • 3)Unmounting Phase :

    • This phase occurs when a component is removed from the DOM. The method "componentWillUnmount" is called here, allowing the component to perform cleanup tasks such as removing event listeners or clearing intervals to prevent memory leaks.

2. What is the difference between state and props?

Answer:

  • Props:
    1. Data is passed from a parent component to a child component.
    2. Data cannot be modified by the child component.
    3. It is controlled by the parent component.
  • State:
    1. Data of the component can be updated locally.
    2. Data can be updated using the setState() method.
    3. It is controlled by the component in which it is declared.

3. What is React Router?

Answer:

  1. Router is a third-party library that can be installed using npm install react-router-dom.
  2. It is used in a single-page application with multiple views.
  3. It is used to navigate from one page to another.
  4. It maps specific components to different URLs, enabling the rendering of distinct UIs for each route.

5.What are the advantages of React?

Answer:

  1. Main advantage of react is "Virtual Dom", a Virtual DOM, a lightweight copy of the actual DOM. This enables react to perform efficient updates by first updating the Virtual DOM, then comparing it to the previous version to identify the necessary changes, and finally updating the real DOM with only the necessary alterations. it updates only where the updation is needed.

Answer:

    • 8.Differentiate between Real DOM and Virtual DOM

    Answer:

      Real DOM:

    1. It's the actual tree-like structure representation of the HTML elements on a webpage.
    2. If we update one component in real dom, it will run from starting of the code and runs everything from first, it will makes the update too slow.

      Virtual Dom:

    1. It's an in-memory, lightweight copy of the Real DOM.
    2. if we update any one component in virtual dom, it will update only the neccessary component and it makes the updates more efficient when compared with real dom.
